coral-glm

This is a very simple e easy to use module. Its goal is to expose all the Vector, Matrices and Quaternion API from the glm C++ library in Lua, following the GLSL syntax.

There are 3 coral components, Vec3, Mat4 and Quat, but they are only for internal use. Therefore, no documentation is available for these types.

The actual API is inside the __init.lua file. This file is the main module file, and returns the table that contains the whole user API of coral-glm. Navigate to any of the provided types`s documentation:

  • Vector 3 API
  • Matrix 4x4 API
  • Quaternion API

As coral-glm follows GLSL syntax, there are no methods, only functions that can receive objects and return results. For efficiency`s sake, the user can optionally pass an object that will be returned, so the function does not need to create a new instance of a coral object e.g.: The function crossVec calculates a cross product between two vectors. It receives two input Vec3 objects, v1 and v2. If these are the only two parameters to crossVec, another Vec3 object will be created inside the function and will be returned as the result. However there is the option to pass a Vec3 object as last argument, so the result will be stored in it. Just a caveat here though, if an object with is not of the matching return type, an error will occur.

Beyond functions, GLSL provides operators for types beside the basic ones, such as matrix and vector products, among others. Coral-GLM also provide most of the operators, only limited by Lua in some cases.

For C++ usage of coral-glm objects, similarly to any native class type, the actual g-truc`s glm API should be used. As glm is a header only library, all of its the source files are embedded into coral-glm and are copied upon installation to /include folder in the same directory as the /modules folder. The user have the option to add the includes manually or to script cmake to find glm/glm.hpp in every directory in the coral path in this way: coral_path_dir/../include/
